यह पृष्ठ वर्णन करता है कि कंटेंट रिपॉजिटरी `@f5-sales-demo/docs-theme` npm पैकेज का उपयोग करके पूरी तरह से ब्रांडेड दस्तावेज़ीकरण साइटें कैसे तैयार करती हैं।

## आर्किटेक्चर

```
┌─────────────────┐
│  Content Repo   │
│  (docs/ folder) │
└────────┬────────┘
         │
         ▼
┌───────────────────────────┐
│   f5xc-docs-builder       │
│   (Docker image)          │
│                           │
│  npm install              │
│  @f5-sales-demo/docs-theme ─────┐  │
│                        ▼  │
│  node_modules/            │
│    @f5-sales-demo/docs-theme/       │
│      ├── config.ts        │
│      ├── index.ts         │
│      ├── fonts/           │
│      ├── styles/          │
│      ├── assets/          │
│      └── components/      │
│                           │
│  Astro Build              │
└─────────────┬─────────────┘
              ▼
┌───────────────────────────┐
│   Astro Build Output      │
│   (static HTML/CSS)       │
└─────────────┬─────────────┘
              ▼
┌───────────────────────────┐
│   GitHub Pages            │
└───────────────────────────┘
```

## बिल्ड प्रक्रिया चरण-दर-चरण

1. **कंटेंट रिपो पुश** — `main` पर पुश (या मैनुअल डिस्पैच) GitHub Pages Deploy वर्कफ़्लो को ट्रिगर करता है
2. **पुनः उपयोगयोग्य वर्कफ़्लो** — कंटेंट रिपो का वर्कफ़्लो बिल्डर को कॉल करता है:
   ```yaml
   jobs:
     docs:
       uses: f5-sales-demo/docs-control/.github/workflows/github-pages-deploy.yml@main
   ```
3. **Docker बिल्डर चलता है** — `f5xc-docs-builder` Docker इमेज में पहले से ही npm के माध्यम से थीम इंस्टॉल होती है। बिल्ड के समय यह:
   - `astro.config.mjs` और `content.config.ts` को `node_modules/@f5-sales-demo/docs-theme/` से Astro प्रोजेक्ट रूट में कॉपी करता है
   - कंटेंट रिपो की `docs/` फ़ाइलों को `src/content/docs/` में कॉपी करता है
4. **Astro बिल्ड** — Astro कॉन्फ़िगरेशन को पढ़ता है जो `createF5xcDocsConfig()` को कॉल करता है। फ़ैक्टरी npm पैकेज स्पेसीफायर्स के माध्यम से सभी थीम एसेट्स को रिज़ॉल्व करती है (जैसे, `@f5-sales-demo/docs-theme/styles/custom.css`)
5. **डिप्लॉय** — बिल्ड की हुई स्टैटिक साइट GitHub Pages पर डिप्लॉय की जाती है

## कंटेंट रिपो की आवश्यकताएँ

एक कंटेंट रिपॉजिटरी को केवल यह चाहिए:

- एक `docs/` डायरेक्टरी जिसमें Markdown (`.md`) या MDX (`.mdx`) फ़ाइलें हों
- एक GitHub Actions वर्कफ़्लो जो बिल्डर को कॉल करे

### न्यूनतम वर्कफ़्लो

```yaml
name: GitHub Pages Deploy
on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  docs:
    uses: f5-sales-demo/docs-control/.github/workflows/github-pages-deploy.yml@main
```

यह वही वर्कफ़्लो है जिसे यह थीम रिपॉजिटरी स्वयं उस दस्तावेज़ीकरण को बिल्ड करने के लिए उपयोग करती है जो आप पढ़ रहे हैं।

## पाथ रिज़ॉल्यूशन

थीम में सभी पाथ npm पैकेज स्पेसीफायर्स का उपयोग करते हैं, जो `package.json` में `exports` मैप के माध्यम से रिज़ॉल्व होते हैं:

```js
// CSS — resolved from node_modules
'@f5-sales-demo/docs-theme/fonts/font-face.css'
'@f5-sales-demo/docs-theme/styles/custom.css'

// Components — resolved from node_modules
'@f5-sales-demo/docs-theme/components/Footer.astro'
'@f5-sales-demo/docs-theme/components/Banner.astro'

// Assets — resolved from node_modules
'@f5-sales-demo/docs-theme/assets/github-avatar.png'
```

बिल्ड वर्कस्पेस में कोई `./theme/` डायरेक्टरी नहीं है। Docker बिल्डर थीम को एक नियमित npm निर्भरता के रूप में इंस्टॉल करता है और Astro `node_modules` के माध्यम से सभी स्पेसीफायर्स को रिज़ॉल्व करता है।

:::note
`font-face.css` के अंदर फ़ॉन्ट फ़ाइल पाथ `./` का उपयोग करते हैं (जैसे, `url("./neusaNextProWide-300.woff2")`)। ये CSS फ़ाइल के सापेक्ष होते हैं, इसलिए वे `fonts/` डायरेक्टरी के अन्य फ़ाइलों को रिज़ॉल्व करते हैं, चाहे पैकेज किसी भी तरह इंस्टॉल हो।
:::

## परिवर्तन प्रसार

थीम परिवर्तन npm पैकेज अपडेट के माध्यम से प्रसारित होते हैं:

1. `@f5-sales-demo/docs-theme` में `main` पर एक परिवर्तन मर्ज होता है
2. Docker बिल्डर इमेज अपडेटेड पैकेज के साथ पुनः बिल्ड होती है
3. अगली बार जब किसी कंटेंट रिपो का वर्कफ़्लो चलता है, तो वह अपडेटेड बिल्डर इमेज का उपयोग करता है
4. Astro बिल्ड अपडेटेड फ़ॉन्ट, स्टाइल, लोगो, घटक और प्लगइन उठाता है
5. कंटेंट रिपो की साइट नई थीम के साथ डिप्लॉय होती है

कंटेंट रिपो को हमेशा Docker इमेज में बंडल की गई नवीनतम थीम मिलती है। यह सभी साइटों में दृश्य एकरूपता सुनिश्चित करता है लेकिन इसका अर्थ है कि थीम परिवर्तनों को मर्ज करने से पहले सावधानीपूर्वक परीक्षण किया जाना चाहिए।
