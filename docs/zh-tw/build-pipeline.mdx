本頁說明內容儲存庫如何使用 `@f5-sales-demo/docs-theme` npm 套件，以產生完整品牌化的文件網站。

## 架構

```
┌─────────────────┐
│  Content Repo   │
│  (docs/ folder) │
└────────┬────────┘
         │
         ▼
┌───────────────────────────┐
│   f5xc-docs-builder       │
│   (Docker image)          │
│                           │
│  npm install              │
│  @f5-sales-demo/docs-theme ─────┐  │
│                        ▼  │
│  node_modules/            │
│    @f5-sales-demo/docs-theme/       │
│      ├── config.ts        │
│      ├── index.ts         │
│      ├── fonts/           │
│      ├── styles/          │
│      ├── assets/          │
│      └── components/      │
│                           │
│  Astro Build              │
└─────────────┬─────────────┘
              ▼
┌───────────────────────────┐
│   Astro Build Output      │
│   (static HTML/CSS)       │
└─────────────┬─────────────┘
              ▼
┌───────────────────────────┐
│   GitHub Pages            │
└───────────────────────────┘
```

## 建置流程逐步說明

1. **內容儲存庫推送** — 推送至 `main`（或手動觸發）會啟動 GitHub Pages 部署工作流程
2. **可重用工作流程** — 內容儲存庫的工作流程會呼叫建置器：
   ```yaml
   jobs:
     docs:
       uses: f5-sales-demo/docs-control/.github/workflows/github-pages-deploy.yml@main
   ```
3. **Docker 建置器執行** — `f5xc-docs-builder` Docker 映像檔已透過 npm 預先安裝主題。在建置時會：
   - 將 `astro.config.mjs` 和 `content.config.ts` 從 `node_modules/@f5-sales-demo/docs-theme/` 複製至 Astro 專案根目錄
   - 將內容儲存庫的 `docs/` 檔案複製至 `src/content/docs/`
4. **Astro 建置** — Astro 讀取設定檔，該設定檔會呼叫 `createF5xcDocsConfig()`。此工廠函式透過 npm 套件指定符解析所有主題資源（例如 `@f5-sales-demo/docs-theme/styles/custom.css`）
5. **部署** — 建置完成的靜態網站部署至 GitHub Pages

## 內容儲存庫需求

內容儲存庫只需具備：

- 包含 Markdown（`.md`）或 MDX（`.mdx`）檔案的 `docs/` 目錄
- 呼叫建置器的 GitHub Actions 工作流程

### 最簡工作流程

```yaml
name: GitHub Pages Deploy
on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  docs:
    uses: f5-sales-demo/docs-control/.github/workflows/github-pages-deploy.yml@main
```

這與本主題儲存庫本身用於建置您正在閱讀之文件的工作流程相同。

## 路徑解析

主題中的所有路徑均使用 npm 套件指定符，透過 `package.json` 中的 `exports` 對應表進行解析：

```js
// CSS — resolved from node_modules
'@f5-sales-demo/docs-theme/fonts/font-face.css'
'@f5-sales-demo/docs-theme/styles/custom.css'

// Components — resolved from node_modules
'@f5-sales-demo/docs-theme/components/Footer.astro'
'@f5-sales-demo/docs-theme/components/Banner.astro'

// Assets — resolved from node_modules
'@f5-sales-demo/docs-theme/assets/github-avatar.png'
```

建置工作區中不存在 `./theme/` 目錄。Docker 建置器將主題作為一般 npm 相依套件安裝，Astro 則透過 `node_modules` 解析所有指定符。

:::note
`font-face.css` 中的字型檔案路徑使用 `./`（例如 `url("./neusaNextProWide-300.woff2")`）。這些路徑相對於 CSS 檔案本身，因此無論套件如何安裝，都會解析至 `fonts/` 目錄內的其他檔案。
:::

## 變更傳播

主題變更透過 npm 套件更新進行傳播：

1. 變更合併至 `@f5-sales-demo/docs-theme` 的 `main` 分支
2. Docker 建置器映像檔以更新後的套件重新建置
3. 下次任何內容儲存庫的工作流程執行時，將使用更新後的建置器映像檔
4. Astro 建置會取得更新後的字型、樣式、標誌、元件及外掛程式
5. 內容儲存庫的網站以新主題完成部署

內容儲存庫始終取得 Docker 映像檔中捆綁的最新主題。這確保了所有網站在視覺上的一致性，但也意味著主題變更在合併前應謹慎測試。
