Dieser Leitfaden dokumentiert die Design-System-Tokens und Komponentenklassen, die in `styles/custom.css` implementiert sind. Er dient als Referenz für Rahmen-, Oberflächen-, Schatten-, Radius-, Schaltflächen-, Verlaufs-, Übergangs-, Fokus- und Typografie-Tokens, die im gesamten Dokumentationsthema verwendet werden.

> **Status**: Alle unten beschriebenen Design-Tokens und Komponentenklassen sind implementiert und einsatzbereit.

## So lesen Sie diesen Leitfaden

Jeder Abschnitt dokumentiert eine Token-Kategorie:

1. **Token-Definitionen** — CSS-benutzerdefinierte Eigenschaften mit Hell-/Dunkel-Modus-Werten
2. **Komponenten-Zuordnung** — welche Dokumentationsthema-Komponenten welchen Token verwenden
3. **Implementierungshinweise** — Verwendungsanleitung und Barrierefreiheitsüberlegungen

Token-Tabellen sind nach **Hellmodus** und **Dunkelmodus** aufgeteilt, wenn sich die Werte unterscheiden.

## 1. Farbsystem

### Alpha-transparente Rahmen

Das Dokumentationsthema verwendet **alpha-transparente Rahmen** anstelle von einfarbigen Grautönen. Dies ermöglicht es den Rahmen, sich natürlich an jeden Hintergrund anzupassen.

```css
/* Alpha-transparentes Rahmenmuster */
border: 1px solid var(--f5-border-default);  /* neutral at 20% alpha */
```

#### Rahmen-Tokens

```css
/* Light mode */
:root[data-theme='light'] {
  --f5-border-faint: #3434341a;    /* --f5-black-3 at 10% */
  --f5-border-default: #34343433;  /* --f5-black-3 at 20% */
  --f5-border-strong: #22222266;   /* --f5-black-4 at 40% */
}

/* Dark mode */
:root {
  --f5-border-faint: #f5f5f51a;    /* --f5-white-2 at 10% */
  --f5-border-default: #f5f5f533;  /* --f5-white-2 at 20% */
  --f5-border-strong: #cccccc66;   /* --f5-white-4 at 40% */
}
```

#### Rahmen-Anwendungsfälle

| Token | Deckkraft | Anwendungsfall |
|---|---|---|
| `--f5-border-faint` | 10% | Dezente Trennlinien, Tabellenzeilen |
| `--f5-border-default` | 20% | Standardrahmen, Kartenumrisse |
| `--f5-border-strong` | 40% | Betonte Rahmen, aktive Zustände |

> **Status**: Abgeschlossen — implementiert in `styles/custom.css`.

## 2. Erhöhungs- und Schattensystem

## 7. Hover-Effekte und Übergänge

### Übergangs-Tokens

```css
:root {
  --f5-transition-fast: 0.15s ease;
  --f5-transition-base: 0.2s ease;
  --f5-transition-bounce: 0.2s cubic-bezier(0.68, -0.2, 0.265, 1.15);
  --f5-transition-decelerate: 0.6s cubic-bezier(0.5, 1, 0.89, 1);
  --f5-transition-spring: 0.2s cubic-bezier(0.54, 1.5, 0.38, 1.11);
}
```

| Token | Dauer | Easing | Anwendungsfall |
|---|---|---|---|
| `--f5-transition-fast` | `0.15s` | `ease` | Die meisten Hover-Zustände, Farbänderungen |
| `--f5-transition-base` | `0.2s` | `ease` | Hintergrundfüllungen, Rahmenänderungen |
| `--f5-transition-bounce` | `0.2s` | `cubic-bezier(0.68, -0.2, 0.265, 1.15)` | Schalter, Umschalter |
| `--f5-transition-decelerate` | `0.6s` | `cubic-bezier(0.5, 1, 0.89, 1)` | Tab-Indikatoren, gleitende Panels |
| `--f5-transition-spring` | `0.2s` | `cubic-bezier(0.54, 1.5, 0.38, 1.11)` | Tooltips, Popover-Einblendung |

### Was animiert werden soll

| Eigenschaft | Sicher zu animieren | Hinweise |
|---|---|---|
| `background-color` | Ja | Standard für Hover-Füllungen |
| `color` | Ja | Textfarbänderungen |
| `border-color` | Ja | Rahmenbetonung beim Hover |
| `box-shadow` | Ja | Erhöhungsänderungen (`will-change` bei Ruckeln verwenden) |
| `transform` | Ja | Skalierung, Verschiebung für Rückmeldung |
| `opacity` | Ja | Ein-/Ausblenden |
| `width`, `height` | Vermeiden | Verursacht Layout-Neuberechnung |
| `padding`, `margin` | Vermeiden | Verursacht Layout-Neuberechnung |

### Komponenten-Übergangs-Zuordnung

```css
/* Sidebar nav items */
nav.sidebar a {
  transition: background-color var(--f5-transition-fast),
              color var(--f5-transition-fast);
}

/* Cards — shadow lift on hover */
.swatch, .icon-card {
  transition: box-shadow var(--f5-transition-base),
              transform var(--f5-transition-base);
}
.swatch:hover, .icon-card:hover {
  box-shadow: var(--f5-shadow-mid);
  transform: translateY(-1px);
}

/* Buttons */
.btn-primary, .btn-secondary, .btn-tertiary {
  transition: background-color var(--f5-transition-fast),
              border-color var(--f5-transition-fast),
              box-shadow var(--f5-transition-fast);
}

/* Links */
a {
  transition: color var(--f5-transition-fast);
}
```

> **Status**: Abgeschlossen — alle fünf Übergangs-Tokens (`fast`, `base`, `bounce`, `decelerate`, `spring`) und Komponentenübergänge implementiert in `styles/custom.css`.

## 8. Fokusring-System

### Fokusring-Tokens

Das Dokumentationsthema verwendet ein **Doppelring**-Muster mit einem inneren Einsatzring und einem äußeren Leuchten:

```css
/* Action focus — using F5 River blue */
:root {
  --f5-focus-action: inset 0 0 0 1px var(--f5-river), 0 0 0 3px var(--f5-river-2);
  /* inset 0 0 0 1px #0e41aa, 0 0 0 3px #6e8dcc */

  --f5-focus-critical: inset 0 0 0 1px var(--f5-red-3), 0 0 0 3px var(--f5-red-2);
  /* inset 0 0 0 1px #a70020, 0 0 0 3px #f06680 */
}

/* Apply to interactive elements */
a:focus-visible,
button:focus-visible,
input:focus-visible,
select:focus-visible,
textarea:focus-visible {
  outline: none;
  box-shadow: var(--f5-focus-action);
}

/* Critical variant for destructive buttons */
.btn-critical:focus-visible {
  box-shadow: var(--f5-focus-critical);
}
```

### Barrierefreiheitshinweise

- Das Doppelring-Muster erfüllt die **WCAG 2.2 Fokusdarstellungs**-Anforderungen (mindestens 2px kontrastierender Umriss)
- Die Verwendung von `box-shadow` anstelle von `outline` ermöglicht die Berücksichtigung des Rahmenradius
- `:focus-visible` stellt sicher, dass der Ring nur bei der Tastaturnavigation erscheint, nicht bei Mausklicks

> **Status**: Abgeschlossen — Doppelring-Fokusmuster (`action`, `critical`) und `:focus-visible`-Stile implementiert in `styles/custom.css`.

## 9. Typografie

### Schriftstapel

```css
:root {
  --sl-font: "proximaNova", system-ui, "Segoe UI", Helvetica, Arial, sans-serif;
  --sl-font-heading: "neusaNextProWide", system-ui, "Segoe UI", Helvetica, Arial, sans-serif;
  --sl-line-height-headings: 1.1;
}
```

### Typskala

| Skala | Größe | Zeilenhöhe | Verwendung |
|---|---|---|---|
| Display 500 | `1.875rem` (30px) | 1,267 | `--sl-text-h1` |
| Display 400 | `1.5rem` (24px) | 1,333 | `--sl-text-h2` |
| Display 300 | `1.125rem` (18px) | 1,333 | `--sl-text-h3` |
| Display 200 | `1rem` (16px) | 1,5 | `--sl-text-h4` |
| Display 100 | `0.8125rem` (13px) | 1,385 | Oberzeilen-Text |
| Body 300 | `1rem` (16px) | 1,5 | Primärer Fließtext |
| Body 200 | `0.875rem` (14px) | 1,429 | Seitenleistentext, Bildunterschriften |
| Body 100 | `0.8125rem` (13px) | 1,385 | Kleingedrucktes |

### Wesentliche Merkmale

| Aspekt | Wert | Hinweise |
|---|---|---|
| Überschriften-Zeilenhöhe | `1.1` | Enge Überschriften für Markenauftritt; `1.2` für bessere Lesbarkeit erwägen |
| Fließtext-Zeilenhöhe | `1.5` (Starlight-Standard) | Gut geeignet zum Lesen |
| Schriftfamilie | Benutzerdefinierte Markenschriften | proximaNova (Fließtext), neusaNextProWide (Überschriften) |
| Überschriftengewichtungen | `h1-h2: 700, h3: 500, h4-h6: 700 Großbuchstaben` | Ausdrücklich für Markenkonsistenz |

> **Priorität**: Keine Änderungen erforderlich. Die aktuelle Typografie ist gut definiert und markengerecht. Eine geringfügige Anpassung der Überschriften-Zeilenhöhe (von `1.1` auf `1.2`) ist optional.
