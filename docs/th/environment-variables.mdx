ธีมจะอ่านตัวแปรสภาพแวดล้อมในช่วงเวลาสร้างจาก `config.ts` และส่วนประกอบที่กำหนดเอง ที่เก็บเนื้อหาจะกำหนดค่าเหล่านี้ใน GitHub Actions workflow เพื่อปรับแต่งแต่ละไซต์โดยไม่ต้องแก้ไขการกำหนดค่า

## อ้างอิงตัวแปร

| ตัวแปร | ค่าเริ่มต้น | วัตถุประสงค์ |
|----------|---------|---------|
| `DOCS_TITLE` | `Documentation` | ชื่อไซต์ที่แสดงในส่วนหัวและแท็บเบราว์เซอร์ |
| `DOCS_SITE` | `https://f5-sales-demo.github.io` | URL ฐานแบบ Canonical สำหรับไซต์ที่ใช้งาน |
| `DOCS_BASE` | `/` | URL ฐาน (เช่น `/my-repo/` สำหรับไซต์โปรเจกต์) |
| `DOCS_DESCRIPTION` | _(สตริงว่าง)_ | คำอธิบายไซต์สำหรับ metadata และปลั๊กอิน llms.txt |
| `DOCS_HOME` | `https://f5-sales-demo.github.io/docs/` | URL หน้าหลักที่ลิงก์จากชื่อไซต์ |
| `GITHUB_REPOSITORY` | _(สตริงว่าง)_ | ใช้เพื่อสร้างลิงก์โซเชียล GitHub และลิงก์แก้ไข |
| `LLMS_OPTIONAL_LINKS` | `[]` | JSON array ของลิงก์เพิ่มเติมสำหรับปลั๊กอิน llms.txt |

## ตำแหน่งที่ใช้ตัวแปรแต่ละตัว

### DOCS_TITLE

```ts
// config.ts
const title = options.title || process.env.DOCS_TITLE || 'Documentation';
```

ส่งต่อไปยังตัวเลือก `title` ของ Starlight และปลั๊กอิน `starlight-llms-txt` ปรากฏในส่วนหัวของไซต์และแท็ก HTML `<title>`

### DOCS_SITE

```ts
// config.ts
const site = options.site || process.env.DOCS_SITE || 'https://f5-sales-demo.github.io';
```

กำหนดคุณสมบัติ `site` ระดับบนสุดของ Astro ใช้สำหรับ canonical URL การสร้าง sitemap และ metadata ของ Open Graph

### DOCS_BASE

```ts
// config.ts
const base = options.base || process.env.DOCS_BASE || '/';
```

กำหนดคุณสมบัติ `base` ระดับบนสุดของ Astro จำเป็นเมื่อปรับใช้ในไดเรกทอรีย่อย (เช่น `https://example.github.io/my-repo/`)

### DOCS_DESCRIPTION

```ts
// config.ts
const description = options.description || process.env.DOCS_DESCRIPTION || '';
```

ส่งต่อไปยังปลั๊กอิน `starlight-llms-txt` เป็นคำอธิบายของไซต์ ใช้ในไฟล์ `llms.txt` ที่สร้างขึ้น

### DOCS_HOME

```ts
// components/SiteTitle.astro
const docsHome = process.env.DOCS_HOME || 'https://f5-sales-demo.github.io/docs/';
```

อ่านโดยส่วนประกอบ `SiteTitle.astro` ที่กำหนดเอง ห่อชื่อไซต์ในส่วนหัวด้วยลิงก์ไปยัง URL นี้ ทำให้ผู้ใช้สามารถนำทางกลับไปยังหน้าหลักของเอกสารส่วนกลางได้ มีประโยชน์เมื่อที่เก็บเนื้อหาหลายแห่งใช้ธีมเดียวกันและคุณต้องการลิงก์ "home" ที่สม่ำเสมอในทุกไซต์

### GITHUB_REPOSITORY

```ts
// config.ts
const githubRepository = options.githubRepository || process.env.GITHUB_REPOSITORY || '';
```

ใช้เพื่อสร้างไอคอนโซเชียล GitHub ในส่วนหัวของไซต์และ URL ฐานของลิงก์แก้ไข GitHub Actions กำหนดตัวแปรนี้โดยอัตโนมัติ (เช่น `owner/repo`)

### LLMS_OPTIONAL_LINKS

```ts
// config.ts
const llmsOptionalLinks = options.llmsOptionalLinks
  || (process.env.LLMS_OPTIONAL_LINKS ? JSON.parse(process.env.LLMS_OPTIONAL_LINKS) : []);
```

JSON array ที่เข้ารหัสของออบเจกต์ `{ title, url }` ที่ส่งต่อไปยังปลั๊กอิน `starlight-llms-txt` สิ่งเหล่านี้จะปรากฏเป็นลิงก์เพิ่มเติมในไฟล์ `llms.txt` ที่สร้างขึ้น

ตัวอย่างค่า:

```json
[{"title": "API Reference", "url": "https://api.example.com/docs"}]
```

## การกำหนดตัวแปรใน GitHub Actions

ที่เก็บเนื้อหาส่งผ่านตัวแปรเหล่านี้ผ่าน workflow ของตน:

```yaml
jobs:
  docs:
    uses: f5-sales-demo/docs-control/.github/workflows/github-pages-deploy.yml@main
    with:
      docs_title: "My Project Docs"
      docs_site: "https://example.github.io"
      docs_base: "/my-project/"
      docs_home: "https://example.github.io/home/"
```

ตัวแปร `GITHUB_REPOSITORY` ถูกจัดเตรียมโดยอัตโนมัติโดย GitHub Actions runner และไม่จำเป็นต้องกำหนดค่าด้วยตนเอง
